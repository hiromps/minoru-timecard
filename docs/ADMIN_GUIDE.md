# ミノルタイムカードシステム 管理者マニュアル

このマニュアルは、ミノル化学工業株式会社の勤怠システム「ミノルタイムカードシステム」を管理する
人事担当者（管理者）向けの操作手順書です。社員の登録、打刻記録の修正、月次集計の確認、データ出力（CSV）などを
画面操作に沿って説明します。

> データ項目の詳細な定義は [DATA_MODEL.md](./DATA_MODEL.md)、セキュリティ設定は [SECURITY.md](./SECURITY.md) を参照してください
> （このマニュアルでは重複説明しません）。

---

## 1. 管理者ログイン

1. ブラウザで管理者用URL（末尾が「**`/admin`**」）を開きます。
2. 「**管理者ログイン**」画面が表示されます。
3. 「**ユーザー名**」と「**パスワード**」を入力します。
4. 必要に応じて「**ログイン情報を記憶する**」にチェックを入れると、次回から入力が省けます
   （共用端末では**チェックを外して**ください）。
5. 「**ログイン**」ボタンを押します。

### デモ環境のログイン情報

デモ（動作確認用）環境では、以下の初期認証情報でログインできます。

| 項目 | 値 |
|------|-----|
| ユーザー名 | `admin` |
| パスワード | `minoru-demo` |

- この値は環境変数 `REACT_APP_DEMO_ADMIN_USER` / `REACT_APP_DEMO_ADMIN_PASS` で上書きできます。
- **この認証情報はデモ専用です。本番環境では使用しないでください。**
  本番環境の認証情報の設定・運用については [SECURITY.md](./SECURITY.md) を参照してください。

ログインに成功すると「**管理者ダッシュボード**」が表示されます。作業を終えたら、右上の「**ログアウト**」ボタンで
必ずログアウトしてください。

## 2. 管理者ダッシュボードの構成

ダッシュボード上部には、4つのタブがあります。用途に応じて切り替えて使います。

| タブ名 | 用途 | 本マニュアルの章 |
|--------|------|------------------|
| **データ出力** | 打刻記録をCSVファイルで出力（初期表示タブ） | 第6章 |
| **月次集計** | 社員ごとの月間集計を表で確認 | 第5章 |
| **社員管理** | 社員の追加・編集・削除、始業/終業時刻の設定 | 第3章 |
| **打刻記録管理** | 個別の打刻記録の修正・削除、ステータス再計算 | 第4章 |

## 3. 社員管理（社員の追加・編集・削除）

「**社員管理**」タブを開きます。登録済みの社員が「社員ID / 氏名 / 部署 / 勤務時間 / 操作」の一覧表で表示されます。

### 3-1. 社員を追加する

1. 画面右上の「**新規社員追加**」ボタンを押します。
2. 「新規社員追加」ウィンドウで、次の項目を入力します。
   - **社員ID**（例: `001`）— 打刻画面での並び順・識別に使われます
   - **氏名**（例: `田中太郎`）
   - **部署**（例: `営業部`）
   - **出勤時間**（所定の始業時刻。初期値 `09:00`）
   - **退勤時間**（所定の終業時刻。初期値 `17:00`）
3. 「**追加**」ボタンを押します。「社員を追加しました」と表示されれば登録完了です。

> **出勤時間・退勤時間**は、その社員の**遅刻・早退・残業の判定基準**になります（[DATA_MODEL.md](./DATA_MODEL.md) 参照）。
> 実際の勤務シフトに合わせて正確に設定してください。

### 3-2. 社員情報を編集する

1. 一覧で対象社員の行にある「**✏️ 編集**」ボタンを押します。
2. 「社員情報編集」ウィンドウで内容を修正します（始業・終業時刻の変更もここで行います）。
3. 「**更新**」ボタンを押します。「社員情報を更新しました」と表示されれば完了です。

### 3-3. 社員を削除する

1. 一覧で対象社員の行にある「**🗑️ 削除**」ボタンを押します。
2. 「〇〇さんを削除しますか？関連する打刻記録も削除されます。」という確認が出ます。
3. 問題なければ「**OK**」を押します。

> **削除するとその社員の打刻記録もすべて削除されます。** 過去の勤怠が必要な場合は、
> 削除前に「データ出力」（第6章）でCSVを保管してください。

### 3-4. 入力時の注意（始業・終業時刻）

- **退勤時間は出勤時間より後**にしてください。逆転していると保存時に
  「退勤時間は出勤時間より後にしてください」と警告が出ます。
- 現在のシステムは**日をまたぐ夜勤には対応していません**。始業・終業は同一日内で設定してください。

## 4. 打刻記録の修正（打刻記録管理）

打刻もれ・打刻ミス・二重打刻などを、管理者が正しい記録に直すための操作です。「**打刻記録管理**」タブを開きます。

一覧には、社員ID/氏名・日付・出勤時刻・退勤時刻・勤務時間・残業時間・ステータス・入力種別（**自動**／**手動**）・操作が表示されます。
**手動**で入力・修正された記録は背景色で区別されます。

### 4-1. 記録を探す（フィルター）

画面上部のフィルターで絞り込めます。

- **日付**: 特定日の記録だけ表示
- **社員ID**: 社員IDの一部を入力して検索
- **手動入力のみ表示**: チェックすると、手動修正された記録だけを表示

うまく表示されないときは「**更新**」ボタンで最新データを再取得してください。

### 4-2. 記録を修正する

1. 対象の記録の行で「**修正**」ボタンを押します。
2. 「**打刻記録の修正**」ウィンドウが開きます。現在の値が初期表示されます。
3. 次の項目を確認・修正します。
   - **社員**: 対象社員（通常は変更不要）
   - **日付**: 記録日
   - **出勤時刻**: 日付と時刻をまとめて指定（必須）
   - **退勤時刻**: 日付と時刻をまとめて指定。まだ退勤していない記録にする場合は
     「**空にする**」ボタンで空欄にできます
   - **修正方法**（下記4-3で解説）
   - **修正理由**（必須。下記4-4で解説）
4. 「**修正実行**」ボタンを押します。「打刻記録を修正しました」と表示されれば完了です。

> **入力チェック**: 退勤時刻が出勤時刻より前の場合や、勤務時間が24時間を超える場合は、
> 保存前に警告が出て実行できません。日付・時刻を正しく指定してください。

### 4-3. 修正方法の選び方（更新 / 削除して再作成）

修正ウィンドウには2つの方法があります。

| 選択肢 | 動作 | 使いどころ |
|--------|------|-----------|
| **既存記録を更新** | 今の記録の時刻だけを書き換えます | 軽微な時刻の直し |
| **削除して再作成（推奨）** | 元の記録を削除し、正しい内容で新しい記録を作り直します | 確実にきれいな記録にしたいとき |

「**削除して再作成**」は内部的に「削除→作成」を**一つの処理としてまとめて実行**するため、途中で失敗して
記録が消えたままになることを防ぎます。迷った場合はこちらを選んでください。

### 4-4. 修正理由の入力

「**修正理由**」欄は**必須**です。初期値として「管理者による時刻修正」が入っていますが、
後から経緯がわかるよう、具体的な理由に書き換えることを推奨します（例:「本人の打刻もれのため、勤務表に基づき入力」）。

修正理由は記録に保存され、CSV出力の「修正理由」列にも反映されます。すべての修正操作は監査ログにも記録されます。

### 4-5. 記録を削除する

1. 対象の記録の行で「**削除**」ボタンを押します。
2. 「社員ID: 〇〇 の 〇〇 の打刻記録を削除しますか？ この操作は取り消せません。」という確認が出ます。
3. 問題なければ「**OK**」を押します。

> 削除は取り消せません。必要に応じて事前にCSVで控えを取ってください。

### 4-6. ステータスの再計算

社員の始業・終業時刻を変更した後などに、既存記録のステータス（遅刻／早退／残業など）を最新の基準で
一括で計算し直せます。

1. 「打刻記録管理」画面右上の「**ステータス再計算**」ボタンを押します。
2. 「全ての打刻記録のステータスを各社員の勤務時間に基づいて再計算します。…実行しますか？」という確認が出ます。
3. 「**OK**」を押すと処理が始まります（件数によっては時間がかかります）。
4. 「ステータスの再計算が完了しました」と表示されれば完了です。

> ステータスやタイムゾーンに関する注意点は [KNOWN_ISSUES.md](./KNOWN_ISSUES.md) を参照してください。

### 4-7. UIで対応できない場合（補足）

このシステムのほとんどの修正はこの画面（打刻記録管理）で完結します。**通常はデータベースを直接操作する必要はありません。**

ごく例外的に、UIでは扱えない特殊なデータ調整や、システム障害時の調査が必要な場合に限り、
Supabase 管理コンソールでの直接操作が選択肢になります。その場合は必ず事前にバックアップ（CSV出力）を取り、
[SECURITY.md](./SECURITY.md) と [KNOWN_ISSUES.md](./KNOWN_ISSUES.md) の注意点（タイムゾーンの扱い等）を確認したうえで、
慎重に実施してください。生SQLでの一括操作は誤操作のリスクが高いため推奨しません。

## 5. 月次集計の見方

「**月次集計**」タブを開きます。社員ごとの1か月の勤怠をまとめて確認できます。

1. 画面上部の「**年**」「**月**」を選ぶと、その月の集計が表示されます。
2. 表には次の列が並びます。

| 列 | 意味 |
|----|------|
| 社員ID / 氏名 | 対象社員 |
| 勤務日数 | その月に勤務（打刻）した日数 |
| 未退勤 | 退勤打刻が無い記録の件数（`⚠️ ○件` と表示。`—` は無し） |
| 総労働時間 | その月の労働時間の合計 |
| 残業時間合計 | 残業時間の合計 |
| 遅刻回数 | 遅刻と判定された日数 |
| 早退回数 | 早退と判定された日数 |

> **「未退勤（⚠️）」が付いている社員**は、退勤打刻もれの可能性があります。第4章の手順で退勤時刻を補ってください。
> 「該当する記録がありません」と出る場合は、その月の打刻がまだ無いか、対象月の選択を確認してください。

## 6. データ出力（CSV）

「**データ出力**」タブを開きます（ログイン直後に表示されるタブです）。

### 6-1. 給与データ出力（25日締め・ワンタップ）

給与の締め日は**毎月25日**です。給与計算用のデータは「**💴 給与データ出力（25日締め）**」から**ワンタップ**で出力できます。

対象期間は自動で **前月26日 〜 当月25日** に設定されます（例: 7月締め = 6月26日〜7月25日）。

1. 画面上部のカードに、対象の締め期間が表示されます（例: 「2026年6月26日 〜 2026年7月25日」）。
   - 既定は直近の締め期間です。**◀ / ▶** ボタンで締め月を前後に変更できます。
2. 「**📥 この締め期間で給与データを出力**」ボタンを押します。
3. `給与データ_YYYY年MM月締め_開始日_終了日.csv` がダウンロードされます。
4. あわせて、画面下部に**不備レポート**が表示されます。

出力CSVは3つのブロックで構成されます（1ファイル・UTF-8 BOM付き）:

- **【明細】**: 日付, 曜日, 社員ID, 社員名, 出勤, 退勤, 勤務時間, **勤務時間(時)**, 残業, 残業(分), ステータス, 直行直帰, 手動入力, **不備**, 修正理由
  - 「勤務時間(時)」は小数（例 `7.50`）で、時給の掛け算にそのまま使えます。
- **【社員別集計】**: 社員ごとの 勤務日数 / 未退勤 / 合計勤務時間 / 合計残業 / 遅刻 / 早退 / 不備件数
- **【不備一覧】**: 給与計算の前に確認・修正すべき項目

### 6-2. 不備の自動チェック（正しく給与時間を把握するために）

出力時に、以下を自動判定します。**「不備一覧」に出た項目は給与計算の前に修正**してください。

| 重要度 | 検出内容 | 意味・対応 |
|---|---|---|
| 🔴 エラー | 退勤打刻がない（未退勤） | 勤務時間を計上できません。「打刻記録管理」で退勤時刻を補完。 |
| 🔴 エラー | 出勤打刻がない | 勤務時間を正しく計上できません。出勤時刻を補完。 |
| 🔴 エラー | 出勤・退勤ともにない | 記録の確認・修正が必要。 |
| 🔴 エラー | ステータスが「設定エラー」 | 時刻の矛盾等。修正のうえステータス再計算。 |
| 🟡 警告 | 勤務時間が0 | 打刻内容を確認。 |
| 🟡 警告 | 勤務時間が16時間超 | 退勤打刻漏れ等の可能性。 |
| 🟡 警告 | 同一日に複数の記録 | 重複の可能性。 |
| 🟡 警告 | 期間内に記録が無い社員 | 欠勤または打刻漏れの可能性。 |

> **エラー**があると給与計算の時間が不正確になります。画面の不備レポートで対象社員・日付を確認し、「**打刻記録管理**」（第4章）で修正 → もう一度出力し直してください。勤務時間は**昼休憩（12:00〜13:00）控除後**の実労働時間です（[DATA_MODEL.md](./DATA_MODEL.md) / [ARCHITECTURE.md](./ARCHITECTURE.md) 参照）。

### 6-3. 任意期間での出力（従来機能）

「**▼ 任意期間で出力（詳細）**」を開くと、開始日・終了日・社員ID（部分一致）を指定した従来のCSV出力ができます（`timecard_records_YYYY-MM-DD.csv`）。すべて空欄なら全件出力です。

- 文字コード: **UTF-8（BOM付き）**。そのまま**Excel**で開いても日本語が文字化けしません。

> CSVは給与計算・監査の元データになります。**社員を削除する前**や、**大きな修正を行う前**に出力しておくと安全です。

## 7. 困ったときは

- ステータスの表示が実態と合わない → 第4章の「ステータス再計算」を実行。改善しない場合は [KNOWN_ISSUES.md](./KNOWN_ISSUES.md) を参照。
- 打刻時刻が9時間ずれて見える等の時刻ズレ → [KNOWN_ISSUES.md](./KNOWN_ISSUES.md)（タイムゾーン）を参照。
- 従業員側の操作案内 → [USER_GUIDE.md](./USER_GUIDE.md) を案内してください。
- セキュリティ・アクセス制限・本番認証情報 → [SECURITY.md](./SECURITY.md) を参照。
